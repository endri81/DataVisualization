# Workshop 7 · Module 8 — Homework
## Case Study: e-Car Data Story

**Due**: Before Workshop 7, Module 10 (Lab)
**Format**: R script + Python script + PDF report (max 8 pages)
**Weight**: Part of Workshop 7 homework (5% of total grade)

### Part A — Big Idea + Storyboard (20 points)
Write a Big Idea statement for an e-Car data story targeting a risk management committee. Create an 8-slide storyboard table: for each slide, state (a) role, (b) assertion headline (<15 words), (c) story type, (d) chart type. The storyboard must follow the story arc: context → complication → mechanism → evidence → recovery → CTA.

### Part B — Hero Chart (25 points)
Produce the hero chart (Slide 2: the rate drop) applying all six annotation techniques: declarative title, grey+accent (spread grey, rate blue), direct labels at endpoints, callout box with curved arrow showing the pp drop, reference line at pre-crisis peak, shaded crisis window. Produce in both R and Python.

### Part C — Mechanism + Tier Slides (30 points)
Produce two additional assertion-evidence slides: (a) the mechanism slide (three-line decomposition: Rate + CoF + Spread, with direct labels and crisis annotation), (b) the tier impact slide (grey+accent on Tier 1, showing it dropped most). Each slide = one assertion headline + one annotated chart. Produce in both R and Python.

### Part D — Netflix vs e-Car Comparison (25 points)
In 300 words, compare the two data stories (M07 Netflix, M08 e-Car) across five dimensions: (a) audience and their decision context, (b) complication and its emotional tone, (c) the mechanism/analytical slide, (d) the forward-looking element, (e) the CTA. Conclude with: which story was harder to build and why?
